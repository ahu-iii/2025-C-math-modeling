# 0006. 问题2：分层 Logistic/计数回归、按药物分组、不建"药物×因素"交互项

## 状态
已接受

## 背景
问题 2 要求分析婚姻状况、既往用药史、初始抑郁程度对疗效/主诉的影响，且题目本身聚焦于"药物自身效果"，未要求跨药物比较因素的调节效应。数据结构中"组别"与"药物"完全一一对应（组1=药物C、组2=药物A、组3=药物B），二者是同一个变量的两套编码，不可同时放入模型（否则设计矩阵秩亏），须择一，本文统一采用药物编码。

> **订正（2026-07-16）**：此前版本曾写"若在回归模型中同时放入'药物'主效应和'药物×因素'交互项，会与'组别'变量完全共线"，这一表述不准确——真正共线的是**组别与药物这两套编码本身**，而非"药物主效应+交互项"这个组合与组别共线。只要不同时放入组别编码，合并模型中的"药物×抑郁程度"交互项在数学上是完全可识别的（已用合并锚定设计矩阵验证：满秩，条件数 71.8）。故本 ADR 不建交互项的依据是**题目要求**（"不考虑因素间的交互作用"），而非"交互项数学上不可估"；这一点在 6.3(5) 的改进方案中已正确表述，此处一并订正背景描述以保持一致。

## 决策
按组（C/A/B）分层建模，每组单独跑一套回归：结局 ~ 婚姻状况 + 既往用药史 + 初始抑郁程度（+ 年龄作协变量）；二分类结局用 Logistic 回归报 OR + 95% CI，计数结局用 Poisson/负二项回归。**不构建"药物×因素"交互项**，通过对比同一因素在三组模型中系数/OR 是否一致，间接观察"因素效应是否因药物而异"。

## 备选方案
- **汇总三组数据、单一模型放入药物主效应 + 因素主效应 + 药物×因素交互项**：并非因数学上不可识别而否决（该交互项本身可以正常估计，见上方订正），而是题目未要求跨药物比较因素的调节效应，且会让模型解释复杂化、在本题脆弱的罕见结局上进一步稀释每参数事件数，故未采纳为主分析；6.3(5) 已将其列为问题四可选的探索性补充分析。
- **仅做单因素卡方/Kruskal-Wallis，不做回归**：无法同时校正多个协变量，作为主分析力度不够，仅保留作为单因素快速探索的辅助工具。

## 影响/后果
分层后每组样本量约为总量的三分之一（约 1000 人），效应量估计的置信区间会比合并建模更宽，需在论文中说明这一权衡；三组回归结果之间不能直接做统计检验意义上的正式比较（只能做描述性对比），若团队后续需要正式检验"效应是否因药物而变"，需另行决定是否补充交互项分析。

## 补充：合并"锚定"模型（实现中新增，2026-07）
问题2 实现（`code/problem2_factors.py`）在分层模型之外，**额外**跑了一个合并三组的 Logistic 模型（自变量含药物主效应 A/B（参照 C）+ 各基线因素主效应，**仍不含药物×因素交互项**），仅用于"非缓解"这一罕见结局。

- **动机**：分层后各组非缓解事件很少（C/A=56、B=37），每参数事件数 EPV 仅 3.7–6.2，药物B 的分层估计尤其脆弱、部分格子近乎完全分离（见诊断表"分离风险类别"）。合并模型事件数 149、EPV≈12.4，可给出稳定的"跨药物平均"因素效应，作为**稳定性锚点**解读脆弱的分层估计（当某分层 OR 明显是分离产物时，以合并估计为准）。
- **与本 ADR 的关系**：分层模型仍是**唯一主分析**；合并模型**不是**主推断，只是稳健性参照。它隐含"因素效应跨药物同质"的假设——这正是分层要规避的——所以绝不能反过来当作分层结论。
- **与 open-Q10 的关系**：open-Q10 默认"不补充正式的跨药物交互检验"，本锚定模型**不违反**该默认（它没有交互项、不做"效应是否因药物而异"的显著性检验），只是一个 main-effects 稳定性参照。
