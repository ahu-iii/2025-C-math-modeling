# ADR 索引

## 什么是 ADR
ADR（Architecture/Any Decision Record，决策记录）是一份"当时为什么这么定"的短记录：不止写"决定了什么"，还写"背景是什么、否定了哪些备选方案、会带来什么后果"。目的是让不在现场的人（比如放假中的队友）也能看懂一个建模决策的来龙去脉，而不需要事后靠聊天记录拼凑。

## 命名规则
`docs/adr/000X-短标题.md`，编号从 `0000` 起零填充、按创建顺序递增，标题用简短中文短语概括决策主题。

## 状态说明
- **已接受**：队内已确认，代码/论文按此执行。
- **待定**：方向已定但仍有分支未拍板，需团队确认（对应分支详见 `docs/建模待决问题.md`）。
- **已弃用**：曾经采用、后被替换的决策，保留记录以防后续讨论重新绕回来。
