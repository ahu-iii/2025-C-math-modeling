# 0005. 症状负担 / 改善程度的定义

## 状态
已接受（数据来源已核实、口径已定；见下方"数据核实结论"）

## 背景
问题 3 要求"症状改善程度"作为连续型疗效指标。D_t 是二元变量（有/无不适），不足以体现"症状越来越轻"这种程度上的变化趋势，需要一个能刻画"不适严重程度"的量化变量。

## 决策
症状负担 B_t = 该时点 **7 类实质性不适症状**（有自杀倾向、副作用导致停药、失眠、脱发、激素水平异常、嗜睡、便秘）中标记为 1 的种类数之和（每个时点算一个计数）；改善程度 ΔB = B_1 − B_12（正值代表改善），可选配合相对改善率 (B_1 − B_12) / B_1。

**"失访"不计入 B_t。** 附件2 虽有 8 类症状栏，但"失访"是缺失/脱落标记而非临床不适症状，与 D_t 的处理保持一致（ADR-0001）；把失访计为"一种症状负担"在临床与统计上都不成立。

> 口径澄清：本决策此前正文与建模方案.md §1.2/§2.5 曾表述为"8 类症状栏之和"，那是对列结构的笼统描述，并非主张失访计入负担。经数据核实（下）后统一改为"7 类实质症状之和、失访排除"，代码（`code/problem2_factors.py` 的 `SUBST`）即按此实现。

## 数据核实结论（open-Q6 已定案）
在问题2 实现中已逐行核对：附件2 现成的**"是否出现不适症状"栏 == 上述 7 类实质症状在该时点标记为 1 的个数之和**，四个时点（1/3/6/12 月）**零误差**。含失访重新计数则与该栏不符（3/6/12 月分别有 26/41/60 行不一致）。这既证明"逐症状重新计数"与"现成汇总栏"两种来源在 7 症状口径下**完全等价**（open-Q6 由此定案，任选其一皆可），也反证**失访本就被数据方排除在负担计数之外**。

## 备选方案（已否决）
把失访计入 B_t（8 类之和）：与附件2 自带的"是否出现不适症状"汇总栏不一致，且将缺失当作症状，语义错误，未采纳。

## 影响/后果
B_t 是问题 3 中唯一的连续/计数型疗效指标，会进入 Kruskal-Wallis/ANOVA 组间比较及熵权-TOPSIS 综合排名的其中一项输入；若后续核实两种数据来源不一致，需要重新计算并在论文中说明差异原因。
