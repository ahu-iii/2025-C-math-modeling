# 0008. 问题4：决策表/决策树做分层选药建议（整合题，不产生新统计）

## 状态
已接受

## 背景
问题 4 要求"结合三药总体评价 + 患者基线特征，为临床医生选药提供建议"。题目性质是综合决策而非新的统计推断——所需信息（总体排名、分层因素效应）均已在问题 2、3 中产出。若在问题 4 里重新跑统计模型，会造成方法重复、结论口径不一致的风险。

## 决策
问题 4 不产生新的统计结果，只做整合呈现：总体层直接引用问题 3 的熵权-TOPSIS 排名给出默认推荐；分层层直接引用问题 2 各组回归中的显著因素，用**决策表**（行 = 患者画像如"初始抑郁程度 × 既往用药史"，列 = 推荐药物 + 依据指标）和/或**决策树**（以基线特征为分裂节点，叶子给推荐药物）呈现，供临床医生查阅。

## 备选方案
- **用机器学习方法（如随机森林、真正训练一棵决策树模型）从数据中学习分裂规则**：统计上更"自动化"，但会引入新的模型和新的过拟合风险，且脱离了问题 2、3 已建立的可解释系数（OR 值），与"整合题不产生新统计"的题目定位不符，未采纳；仅借用"决策树"这一呈现形式（人工整理规则），不做机器学习训练。
- **只给总体排名、不做分层推荐**：无法回应题目"结合基线特征"的要求，信息量不足，未采纳。

## 影响/后果
决策表/决策树的规则来自问题 2 中显著性未必很强的系数，需要在论文中明确标注哪些分层建议证据较强、哪些只是描述性提示，避免给出过度自信的临床建议；同时须重申题目已排除的因素（用药量、价格、药物交互）不在建议范围内。
