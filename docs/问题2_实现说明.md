# 问题2：基线因素与不适主诉代理结局 —— 实现说明

> 代码：`code/problem2_factors.py`
> 运行：`uv run python code/problem2_factors.py`
> 本文档说明技术口径与当前结果，不是论文正文。

## 一、分析对象

问题2考察婚姻状况、既往用药史和初始抑郁程度与后续结局的关系。附件2没有随访期抑郁评分，因此代码中的“非缓解”实际定义为：

```text
D12=1，即第12月仍记录到不适主诉
```

它是主诉代理结局，不等同于临床抑郁未缓解。其余两个结局为“任一时点有主诉”和四时点症状计数总和。

## 二、数据处理

### 2.1 左连接全队列

附件2只记录出现过不适的受试者；未出现在附件2中的人按题面解释为“未出现不适”，不是随访数据缺失。因此以附件1的3149人为基准左连接附件2，未连接的症状标记补0。

### 2.2 重复记录

附件2二院药物A组有194名受试者重复录入，其中159对完全一致、35对存在局部差异。按 `(医院, 序号, 组别)` 对症状标记取并集合并。与保留首行相比，只改变1人的第12月代理结局和2人的自杀倾向标记，不影响总体结论。

### 2.3 样本量说明

建模队列为C=1049、A=1050、B=1050。题面“每组525例”与两院附件合并后的实际行数存在口径差异，代码按附件实际记录分析；二院C组少1例是原始数据异常，未自行补造。

## 三、模型

| 结局 | 发生水平 | 方法 |
|---|---:|---|
| 第12月仍有主诉（代码名“非缓解”） | 37—56例/组 | 分层Firth Logistic；合并Logistic作稳定锚 |
| 任一时点有主诉 | 约32% | 分层普通Logistic |
| 总症状负担 | 计数 | 分层负二项回归 |

自变量为婚姻状况、既往用药、初始抑郁程度和年龄。初始抑郁程度主模型按轻/中/重编码为1/2/3；另以分类变量重新拟合合并模型，核查线性等级假设。

## 四、多重比较口径

结果表共86个系数检验。当前实现明确区分两种校正族：

- **主口径：全表86项统一BH-FDR**，字段为 `p_FDR`、`显著_FDR`；
- **敏感性：每个模型内部BH-FDR**，字段为 `p_FDR_模型内`、`显著_模型内FDR`。

全局FDR主口径下只有1项保持显著：合并模型中初始抑郁程度每加重一级，发生第12月主诉的OR=2.065，95%CI 1.399—3.050，原始p=0.0003，全局FDR p=0.0258。

模型内校正仍得到2项：上述合并效应，以及B组的等级斜率OR=3.509、模型内FDR p=0.0126。后者未通过全局FDR，因此只作敏感性结果，不再写成独立的稳健结论。

## 五、抑郁程度编码敏感性

把轻/中/重改为分类变量后，合并模型中真实严重度两项的联合Wald检验为2自由度；未知状态由单独缺失指示项吸收，不进入严重度联合检验。

该敏感性分析用于回答：主模型的OR=2.065是否只是“每升一级等距”的编码假设造成。结果写入：

- 严重度两项联合 Wald：χ²=12.615，df=2，p=0.001822；
- 中度 vs 轻度：OR=2.448，95%CI 0.990—6.054；
- 重度 vs 轻度：OR=4.792，95%CI 1.789—12.835。

分类编码仍显示总体关联，但中度单项区间跨1，因此主模型的线性等级OR应解释为简洁趋势参数，而不是三个等级间严格等距已经得到证明。完整结果写入：

```text
output/problem2_severity_encoding_sensitivity.csv
```

## 六、结果边界

- 可以说：基线抑郁程度与第12月仍有不适主诉存在稳健关联。
- 可以说：婚姻状况和既往用药史没有通过当前多重比较主口径。
- 不能说：抑郁程度已经被证明影响临床抑郁缓解；随访结局没有抑郁量表。
- 不能说：B药对重度患者更差；重度亚组事件少且没有正式的药物×程度交互检验。
- 不显著表示证据不足，不表示因素没有作用。
