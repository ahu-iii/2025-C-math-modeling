# 抗抑郁药物疗效问题 —— 总体建模方案（Methodology）

> 适用范围：问题 1 ~ 问题 4 的整体建模思路、变量与指标定义、方法选择及理由。
> 定位：这是队伍的"方法总纲"，先统一口径再各自写代码；论文的"模型建立与方法"部分可直接基于本文档扩写。
> 关联文档：问题 1 的具体实现见 `docs/问题1_实现说明.md`，问题 2 见 `docs/问题2_实现说明.md`。

## 〇、一句话总纲

**问题 1 显示三组在年龄、既往用药史和初始抑郁程度上的差异很小；婚姻状况总体卡方主要由 B 组70人的集中缺失驱动，剔除未知后 p=0.133、Cramér's V=0.040。该模式提示组间差异性缺失或记录流程异常，但不能仅凭现有数据判定为 MNAR。** 后续药物比较仍须报告不确定性，不能把“基线检验不显著”写成因果归因已经得到证明。

**问题 2 进一步印证了这条主线：** 在婚姻状况、既往用药史、初始抑郁程度、年龄这几个基线因素里，经多重比较（FDR）校正后**只有"初始抑郁程度"独立影响疗效——基线越重越难缓解**（合并模型 OR≈2.07，抑郁程度每加重一级、未缓解风险约翻倍；药物 B 组更强，OR≈3.5），婚姻状况/既往用药史/年龄均不再显著。换言之，疗效由**病情本身与药物**驱动，人口学基线因素影响有限，与"三组可比"彼此呼应。

四个问题的关系是层层递进的：

```
问题1  基线是否可比？        —— 描述性统计 + 假设检验（已完成）
   │   （建立"三组可比"的前提）
问题2  哪些基线因素影响疗效？ —— 回归 / 分层关联分析（已完成）
   │   （结论：仅"初始抑郁程度"独立影响疗效，为问题4分层推荐做准备）
问题3  三种药到底哪个好？     —— 纵向重复测量 + 综合评价
   │   （给出 A/B/C 的总体排名）
问题4  临床怎么选药？         —— 综合决策（不产生新统计，是问题2+3的整合）
```

## 一、数据与变量总览

### 1.1 附件1（基线数据，横截面）

每位受试者一行，one-hot 编码。经问题 1 清洗后得到的规范变量：

| 变量 | 类型 | 取值 | 角色 |
|---|---|---|---|
| 组别 | 分类 | 1=药物C(对照) / 2=药物A / 3=药物B | 分组变量 |
| 年龄 | 连续 | 岁 | 基线协变量 |
| 婚姻状况 | 分类 | 未婚/已婚/离异/丧偶/未知 | 基线协变量（B组"未知"偏多，需注意缺失） |
| 既往抗抑郁药使用情况 | 分类 | 无/使用过/其它 | 基线协变量 |
| 初始抑郁程度 | 有序分类 | 轻度<中度<重度 | 基线协变量（有序，可赋 1/2/3） |
| 医院 | 分类 | 一院/二院 | 均衡性核查用（问题1已验证均衡） |

### 1.2 附件2（随访主诉，纵向 / 重复测量）

宽表结构：每位受试者一行，列 = 8 类不适症状 × 4 个随访时点（第 1、3、6、12 月）。

- **8 类症状栏**：失访、有自杀倾向、副作用导致停药、失眠、脱发、激素水平异常、嗜睡、便秘。
- **取值**：空 = 该时点未出现该症状；`1` = 出现该症状。
- **"是否出现不适症状"栏**：是对前面各症状栏在每个时点的综合统计（每个时点一个数字，反映该时点主诉症状的数量/是否出现）。
- 随访只记录"出现不适"的受试者，**感受良好者不予记录 → 空值同时意味着"没记录"和"没症状"，二者在建模时按"该时点无不适"处理**（题目已明确"空格表示未出现相应不适症状"）。
- **失访 / 数据缺失**：附件2 单独有"失访"标记；题目在问题3 明确"不需要考虑数据缺失和失访对评估的影响"，故问题3 可按现有记录直接计算，但要在局限性里说明。

**关键提醒**：附件2 的原始数据里没有现成的"疗效"变量，疗效必须由我们自己从症状标记里**构造**出来。这是全局最重要的一步（见第二节）。

## 二、核心操作性定义（全队必须统一，后续所有问题复用）

> 这一节是整份方案的"地基"。问题 2、3、4 的所有结论都依赖这几个定义，**务必先在队内敲定并在论文里明确写出、给出理由**，避免各人代码口径不一致导致结果对不上。

### 2.1 "不适 / 主诉"（单时点）

对某受试者在某时点 $t \in \{1,3,6,12\}$ 月：若该时点任一实质性不适症状（失眠、脱发、激素水平异常、嗜睡、便秘、有自杀倾向、副作用导致停药中至少一项）标记为 `1`，则 $D_t = 1$（该时点有不适），否则 $D_t = 0$。

- 建议把"失访"单独处理，不计入 $D_t$（它是缺失，不是症状）。
- 也可直接采用附件2 现成的"是否出现不适症状"栏作为 $D_t$ 的来源，两种口径二选一并说明。

### 2.2 "适应期"（题目特别定义，务必落实）

题目原文：受试者前期（第 1、3 月）出现不适、但后期（第 6、12 月）不适消失，视为已适应该药物，**此时前三个月的不适不计入药物质量问题**。

形式化：若 $D_1$ 或 $D_3 = 1$，且 $D_6 = D_9\text{...}$（此处为 $D_6 = D_{12} = 0$），则判定为"适应期反应"，在评价药物质量/疗效时对该受试者的早期不适做豁免处理。**这是本题区别于普通临床数据分析的关键规则，评委会重点看有没有落实它。**

### 2.3 第12月无主诉率（代理缓解率）

- **个体层面**：主口径定为 $D_{12}=0$（第12月未记录不适主诉）。
- **组层面**：某组满足该条件的人数 / 该组总人数。
- **解释边界**：附件2没有随访抑郁量表，该指标是主诉代理终点，不是临床抑郁缓解率。

> 两点口径 $D_6=D_{12}=0$ 保留为定义敏感性分析。它与“第6或12月仍有主诉”互为补集；若两点口径进入 TOPSIS，必须删除后者，不能把同一信息的正反两面重复计权。综合指标的安排不反向决定终点定义。

### 2.4 "复发 / 复发率"（Relapse）

- **个体层面**：原本已缓解（某中间时点症状消失），后期又出现症状。判据：$D_3 = 0$ 或 $D_6 = 0$（曾缓解，**宽松版已定案**）**且** $D_{12} = 1$（末次又出现）。
- **复发率**：复发人数 / 曾缓解人数（分母是缓解者，不是全体，务必注意）。**并列报告"全体"口径作辅助描述指标。**

> ✅ **曾缓解取宽松版（ADR-0004）**：缓解既已定为"某**单个**时点无不适"，曾缓解就该是同一把尺子量更早的窗口（第 3、6 月）。严格版（仅 $D_6=0$）等于对"曾缓解"用比"缓解"本身更严的标准——第 12 月的单次无不适算数、第 3 月的不算数，此不对称无依据。
>
> 12组定义组合中，B在当前样本的固定权重扫描里均呈排名第一倾向。该检查说明结果方向不由某一个定义选择驱动，但不包含受试者抽样误差。

### 2.5 主诉负担及随访期变化

- **症状负担**：$B_t$ = 受试者在时点 $t$ 的不适症状种类数（对 **7 类实质症状**在该时点求和，**失访不计入**——它是缺失而非症状）。已核实：附件2"是否出现不适症状"栏就等于这 7 类之和（四时点零误差），两种来源等价，直接用现成栏亦可。详见 ADR-0005、已定案的待决问题第 6 条。
- **随访期变化**：$\Delta B = B_1-B_{12}$。第1月已经是治疗后随访，因此它不是治疗前后改善量，正值只表示第1月至第12月记录的主诉数下降。

### 2.6 "不良反应 / 安全性"指标

副作用发生率、因副作用停药率、失访率、以及"有自杀倾向"发生率（严重不良事件），按组分别统计。这些进入问题3 的综合评价与问题4 的安全性权衡。

## 三、问题 1：基线特征分析（已完成，此处仅归档）

**目标**：三组基线（年龄、婚姻状况、既往用药史、初始抑郁程度）分布是否存在显著差异。

**方法**：
- 连续变量（年龄）：正态性（Shapiro-Wilk）+ 方差齐性（Levene）诊断 → 满足则 ANOVA，否则 Kruskal-Wallis；显著再做事后两两比较。
- 分类变量：卡方独立性检验（期望频数过小时用蒙特卡洛置换检验兜底）+ Cramér's V 效应量。
- 多重比较：Benjamini-Hochberg（FDR）校正。

**结论**：年龄、既往用药史、初始抑郁程度未观察到显著组间差异。婚姻状况表面显著（χ²=152.871, p<0.001），但总卡方的91.5%由“未知”一列贡献；剔除未知后 χ²=9.810, p=0.133，Cramér's V降至0.040。该结果定位了**组间差异性缺失**，但不识别具体缺失机制。详见 `docs/问题1_实现说明.md` 第3.4节。

## 四、问题 2：基线因素对疗效与主诉的影响（关联 / 推断）（已完成）

**目标**：探讨婚姻状况、既往用药史、初始抑郁程度对疗效指标与主诉的影响，刻画因素与结局的关系。

**题目约束**：每组对应一种药物，**分析聚焦药物自身效果、不考虑因素间交互 → 建议按组分层（stratify by drug）分别建模**，避免引入 药物×因素 交互项。

**建模步骤**：

1. **先定结局变量**（用第二节定义）：可同时分析
   - 二分类结局：是否缓解（$D_{12}=0$）、是否复发、是否出现某类主诉；
   - 连续/计数结局：症状负担 $B_t$、改善程度 $\Delta B$。

2. **建模方法（按结局类型选）**：
   - 二分类结局 → **Logistic 回归**：结局 ~ 婚姻状况 + 既往用药史 + 初始抑郁程度（+ 年龄作协变量）。报告 **优势比 OR 及 95% 置信区间**，解释每个因素独立的影响方向与强度。
   - 有序结局（如按抑郁程度分层的疗效等级）→ 有序 Logistic（比例优势模型）。
   - 计数型症状负担 → Poisson 或负二项回归（过离散时用负二项）。
   - 若只想做单因素快速检验：分类因素 × 二分类结局用卡方/Fisher，因素 × 连续结局用 Kruskal-Wallis —— 但回归能同时校正其他因素，作为主分析更稳。

3. **每组（C/A/B）各跑一套**，描述同一因素在不同药物组中的关联方向；未经正式交互检验，不把分层系数差异解释为药物效应异质性，也不直接生成分人群推荐。

4. **报告规范**：给效应量 + 置信区间 + 校正后 p 值，不要只堆 p 值；对婚姻状况要延续问题1 的缺失说明（B 组"未知"较多）。

**实现要点（已落地，详见 `docs/问题2_实现说明.md`）**：结局从附件2 构造，须以附件1（3149 人）左连接附件2 并对缺席者记"全程无不适"；附件2 二院组2 有整块重复录入，已按并集去重（ADR-0010）。非缓解是罕见结局（各组 37–56 例、EPV 3.7–6.2、B 组有完全分离格子），故用 **Firth 惩罚 Logistic + 惩罚 profile-likelihood 置信区间**，并以含药物主效应的**合并模型作稳定性锚点**（ADR-0006 补充）；高发的"任何主诉"用普通 Logistic、计数型"症状负担"用负二项；全部检验用 BH-FDR 校正。

**结论**：以全部86个系数为一个检验族做全局 BH 校正后，仅合并模型中的初始抑郁程度保留统计信号（OR 2.07，95%CI 1.40–3.05，原始 p=0.0003，全局 FDR p=0.0258）。这说明基线严重程度与第12月仍有主诉相关；它不是抑郁疗效的直接证据，也不能据分层模型的点估计差异推出某药更适合重度患者。模型内 FDR 仅作为敏感性结果。

## 五、问题 3：三种药物疗效综合评价（纵向 + 综合评价，核心问题）

**目标**：比较第1—12月随访期内主诉变化，综合评估 A、B、C 三种药的主诉代理终点。附件2没有治疗前抑郁评分，因此不把第1月称为用药前基线。

**这是全题的核心，也是唯一充分利用"4 个时点纵向结构"的地方。** 分两层做：

### 5.1 指标层：逐一计算并做组间检验

按组（C/A/B）分别计算第二节定义的各项指标，并检验组间差异：

| 指标 | 计算 | 组间检验 |
|---|---|---|
| 第12月无主诉率（代理缓解率） | D12=0 人数 / 总人数 | 卡方检验（3 组比例） |
| 复发率 | 复发人数 / 缓解人数 | 卡方检验 |
| 症状改善程度 $\Delta B$ | 平均 $B_1-B_{12}$ | Kruskal-Wallis / ANOVA |
| 副作用发生率、停药率、失访率、自杀倾向率 | 各自比例 | 卡方检验 |

**务必在计算缓解/改善时落实"适应期豁免"规则**（2.2），这是本题得分点。

### 5.2 轨迹层：重复测量模型（随访期变化）

因每人有 1/3/6/12 月共 4 次观测，观测非独立，单纯逐时点比较会低估相关性。推荐：

- **广义估计方程（GEE）**：$P(D_t=1) \sim 药物 + 时间 + 药物\times时间$，交互项检验不同药物的随访主诉轨迹是否不同。显著的时间项不能单独归因于药物，因为缺少治疗前同尺度结局。
- **探索性生存分析**：仅在预先说明的早期主诉子组中分析“首次无主诉”，并把首次失访作为右删失；该后验子组分析不能当作全队列主结论。

### 5.3 综合层：多指标合成一个总排名（ADR-0012 定案）

各指标量纲/方向不同（缓解率越高越好、复发率越低越好、豁免后持续不适率/自杀倾向率越低越好），需综合成单一评价。指标集剪枝为 4 项，每项对应一条独立的临床轴：

| 指标 | 方向 | 轴 |
|---|---|---|
| 第12月无主诉率（D12=0） | 效益型 | 主诉代理终点 |
| 复发率（/曾缓解） | 成本型 | 稳定性 |
| 豁免后持续不适率（D6=1 或 D12=1） | 成本型 | 药物质量 |
| 自杀倾向率 | 成本型 | 安全红线 |

出局的两项：**ΔB 均值**因基线依赖出局（变化量会奖励基线差的组，方向与"疗效好"相反）；**末次症状负担 B12** 因与"缓解率 = 1−P(D12=1)"同为一个 D12 的两种写法而出局（TOPSIS 假定各准则正交，同轴指标会被重复计数）。ΔB 仍保留在 5.1 的描述性表格中，只是不进综合排名。

**主口径：等权 + TOPSIS。** 等权外生于 3×k 矩阵、透明且不可调参，不引入"谁定的这个权重"的口子。

**熵权法降级为敏感性对照**，须在论文中同时报告其失效：熵权法度量的是各指标在 3 个备选间的相对离散度，n=3 下估不出有意义的离散度——同一指标「缓解率」在熵权两种通行预处理（原始值口径 vs 极差归一口径）下权重相差 423 倍（0.0011 vs 0.4532），题目正文点名的头号指标数值上等于被从模型中删除。两种口径都实现、都报告。

**两类不确定性必须分开**：Dirichlet 全空间扫描（20,000组）固定当前样本指标，只回答权重变化是否改名次，B第一占99.9%；受试者分层 bootstrap 同时重算指标，B第一为92.5%，且 B−A 贴近度差的95%区间包含0。前者不是统计置信度。

**产出**：固定样本下的权重敏感性与受试者bootstrap抽样不确定性。当前结论口径为“B在所选指标下呈较一致排名优势，但相对A仍有抽样不确定性；A/C不给确定次序”，不得写成已证实优效。详见 `docs/问题3_实现说明.md`。

## 六、问题 4：临床选药建议（综合决策，不产生新统计）

**目标**：结合三药总体评价 + 患者基线特征，为临床医生选药提供建议。

**定位**：这是**整合题，不是新建模**。把问题3 的总体排名与问题2 的"哪种人群更适合哪种药"结合，输出一套**可操作的决策规则**。

**做法**：

1. **总体层**：基于问题3 的综合排名给出默认推荐（含疗效—安全性权衡，例如缓解率与副作用/自杀倾向率的取舍）。
2. **分层层**：基于问题2 的分组回归结果，按基线人群细化。可用两种呈现：
   - **推荐决策表**：行 = 患者画像（如 初始抑郁程度 × 既往用药史），列 = 推荐药物 + 依据指标；
   - **决策树**：以基线特征为分裂节点，叶子给推荐药物，直观易向医生解释。
3. **约束与局限**：明确题目排除因素（不考虑用药量、价格、交互）；说明数据局限（B 组婚姻缺失、失访）；给出使用该建议的边界条件。

**产出**：一句话总体推荐 + 分层推荐表/决策树 + 依据说明。

## 七、方法选择速查表

| 场景 | 推荐方法 | 理由 |
|---|---|---|
| 3 组比例（率）比较 | 卡方检验 / Fisher | 分类结局标准做法 |
| 3 组连续变量比较 | ANOVA（正态齐性）/ Kruskal-Wallis（否则） | 与问题1 口径一致 |
| 多因素对二分类结局 | Logistic 回归（报 OR+CI） | 可同时校正多个因素 |
| 多因素对计数结局 | Poisson / 负二项回归 | 症状计数、可处理过离散 |
| 4 时点重复测量 | GEE / 混合效应模型 | 处理同一人多次观测的相关性 |
| 达到缓解/复发的速度 | Kaplan-Meier + Log-rank | 利用时间信息，处理删失 |
| 多指标综合排名 | 熵权法 + TOPSIS（+敏感性分析） | 客观赋权、结论稳健 |
| 分层选药建议 | 决策表 / 决策树 | 可解释、易向临床沟通 |
| 多重比较 | Benjamini-Hochberg (FDR) | 控制假阳性，与问题1 一致 |

## 九、分工建议（谁负责哪部分）

**核心认识：建模 = 决策（做什么、为什么），编程 = 执行（具体怎么算）。二者不是两个割裂的环节，而是同一件事的两个时刻——代码没法在建模决策没定之前写出来。** 所以下面的分工不是"各干各的互不相关"，而是"同一条链上的接力"。

### 9.1 三类工作的本质区别

| 环节 | 回答的问题 | 产物 | 谁最适合牵头 |
|---|---|---|---|
| 建模 | 结局变量是什么？做什么假设？该用回归/生存/TOPSIS？缓解/复发怎么定义、为什么？ | 本方案这样的"方法总纲" | 编程者牵头定"能落地的定义"，全队参与判断 |
| 编程 | 怎么清洗数据、跑方法、出数字和图？ | 代码 + `output/` 里的表和图 | 编程者（你） |
| 写作 | 为什么这个方法合理？结果的临床含义是什么？ | 论文"模型建立/求解/结果分析"章节 | 非编程队友 |

### 9.2 具体到本题的分工

**编程者（你）负责——因为这些必须"具体到能写进代码"，谁写代码谁先撞上模糊点：**
- 第二节的**操作性定义**落地成代码常量（不适、适应期、缓解、复发、症状负担）。
- **方法选择**（第七节速查表）：结局是二分类就 Logistic、是计数就负二项、4 时点就 GEE……这些选择本身就是建模，但只有编程者能确认"这个方法在这份数据上真的跑得通"。
- 数据清洗、跑模型、出图出表，存到 `output/`。

**非编程队友负责——这些是"建模思维"但完全不需要写代码，是论文的主体字数：**
- **方法的理由与文献**：为什么用 Logistic 回归 / TOPSIS / 生存分析，缓解率/复发率的临床定义与出处，说明这些是标准做法（查文献、写进论文）。
- **假设检查与结果解读**：拿到 OR、p 值、排名后，解释它的临床含义、判断假设是否成立。这是建模，但不碰代码。
- **论文"模型建立"章节的撰写**：把你定的定义和方法，扩写成正式的模型叙述。

**全队一起做（关键动作）：**
- **坐下来一起锁定第二节的操作性定义**。"缓解 = 第12月无不适是否合理？"这类问题是**判断题不是数学题**，非编程队友完全能参与、也应该参与——这一步定了，建模就完成了 80%，剩下的才真的是编程。

### 9.3 交接节奏（按问题推进）

1. **问题2 前**：全队开会锁定第二节定义 → 你写清洗+回归代码 → 队友查文献写"为什么这么建模" + 解读 OR。
2. **问题3**：你算各率（落实适应期豁免）+ 跑 GEE + TOPSIS → 队友写指标的临床解释 + 综合评价方法的合理性论证。
3. **问题4**：你把 2、3 的结果整理成决策表/决策树 → 队友写选药建议的临床叙述 + 局限性。
4. **贯穿全程**：每问回扣"问题1 已证明基线可比"这条主线；你出图表，队友把图表织进论文。

**一句话**：你负责"把模型定到能算、并把它算出来"，队友负责"论证这个模型为什么对、并解释算出来的东西意味着什么"。建模不是某一个人的活，是你把定义和方法钉死、队友把理由和解读补上的**合作产物**。
