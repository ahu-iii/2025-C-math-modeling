## 一、摘要（30秒版）

**问题1问的是：** 药物C（对照组）、药物A、药物B这三组受试者，入组时的基本情况（年龄、婚姻状况、既往用药史、初始抑郁程度）是否本来就存在差异？

**为什么这一步必须先做：** 问题2、3要比较三种药物的疗效好坏。但如果三组人一开始底子就不一样（比如某一组本来病情更重、年纪更大），那后面看到的"疗效差异"就可能是人群本身的差异造成的，而不是药物的问题。这一步就是排除这个干扰。

**结论：**

| 基线变量 | 三组之间有无显著差异 | 说明 |
|---|---|---|
| 年龄 | **无差异** | 三组年龄分布几乎一致 |
| 既往用药史 | **无差异** | 三组用药史构成一致 |
| 初始抑郁程度 | **无差异** | 三组病情轻重构成一致 |
| 婚姻状况 | **有差异（需要说明原因）** | 差异主要来自"药物B组有70人婚姻状况缺失"，其他三组分布本身很接近 |

**一句话结论：** 三组基线基本可比，年龄/用药史/病情三个维度可以放心地认为三组人群是均衡的；婚姻状况的"差异"是数据缺失分布不均导致的，不是真实的人群差异，报告里需要单独说明这一点，不能直接写"婚姻状况三组不同"。

## 二、数据说明

数据来自附件1（两个医院合并后共 3149 人：一院1574人 + 二院1575人，理论上应各525人×3组×2医院=3150人，二院组1实际只有524人，少1人，属于数据本身的情况，不是我们处理错了）。

附件1里每个受试者一行，关键列是"one-hot"编码（下面举例说明，方便大家看代码时理解）：

```
婚姻状况：未婚 已婚 离异 丧偶   ← 4列，命中的那一列填1，其余空着
既往抗抑郁药使用情况：无 使用过抗抑郁药 其它   ← 3列
抑郁程度：轻度 中度 重度   ← 3列
```

代码里第一步做的事情，就是把这种"4列拆成1列"的编码，还原成 1 个正常的分类变量（比如把"未婚=1,已婚=空,离异=空,丧偶=空"这一行还原成"婚姻状况=未婚"）。**如果4列全是空的（说明这项没记录/失访），我们没有直接删掉这个人，而是记成"未知"这个类别**，这样既不会丢样本量，又能如实反映缺失的情况。

## 三、方法怎么选的（给想看细节 / 要写论文方法部分的同学）

### 3.1 年龄（连续变量）——三组比不比得动？

年龄是数字，要比较3组的均值/分布是否一样，常规套路是"方差分析 ANOVA"，但ANOVA有个前提：每组数据要接近正态分布，且各组方差要差不多。所以流程是：

1. 先用 **Shapiro-Wilk 正态性检验** 看每组年龄是否正态
2. 再用 **Levene 方差齐性检验** 看三组方差是否相近
3. 如果两个前提都满足 → 用 **单因素方差分析 ANOVA**
4. 如果任一前提不满足 → 改用不要求正态分布的 **Kruskal-Wallis 检验**（ANOVA的"非参数版"，比较的是排序后的分布而不是均值本身）

> 我们这批数据每组n=1000+，Shapiro检验在这种大样本下几乎总会判定"不服从正态"（这是这个检验本身的已知特性，样本量一大就很敏感），所以代码自动切换到了 Kruskal-Wallis，结果 p=0.637（远大于0.05），**说明三组年龄分布没有显著差异**。

如果结果显著，代码里还准备了事后两两比较（ANOVA显著就用Tukey HSD，Kruskal-Wallis显著就用Mann-Whitney U两两比较+Bonferroni校正），但这次年龄没有触发，用不上。

### 3.2 婚姻状况 / 既往用药 / 抑郁程度（分类变量）——用卡方检验

这三个都是"组别 × 类别"的列联表问题（比如"3组 × 婚姻状况4类"），标准方法是 **卡方独立性检验（chi-square test of independence）**：检验"落入哪个类别"跟"属于哪一组"是不是没关系（原假设：无关，即三组分布一致）。

卡方检验有个使用条件：列联表里每个格子的**期望频数不能太小（一般要求≥5）**，否则近似不准。代码里对每个变量都检查了这一点，如果有格子期望频数<5，会额外跑一个 **蒙特卡洛置换检验**（把组别标签随机打乱2000次，看真实卡方值排第几）作为交叉验证，这样即使卡方检验的前提不完全满足，结论也站得住。

### 3.3 多重比较校正

我们一次性测了4个基线变量（年龄、婚姻、用药史、抑郁程度），每个都用 α=0.05 的标准判断"显著"，但测的次数一多，本来不显著的东西"蒙对"显著的概率也会累积升高（多重比较问题）。所以代码额外做了一次 **FDR (Benjamini-Hochberg) 校正**，把4个p值一起调整，校正后的显著性判断更可靠，这也是我们最终结论表里用的判断标准。

### 3.4 婚姻状况显著的原因排查

卡方检验发现婚姻状况在三组间显著（p<0.001），但看列联表（`output/test_results.csv`）会发现：药物C组和药物A组的"未知"人数都是0，唯独药物B组有70个人婚姻状况缺失。把"未知"这一类拎出来看，就是它在拉高卡方值，其余"未婚/已婚/离异/丧偶"四类在三组里的比例其实很接近。**这提示婚姻状况这一项的缺失可能不是随机的（missing not at random），而是集中出现在B组的数据里**，这是数据本身的特点，需要在报告里作为局限性说明，而不是说"三组患者婚姻状况本身不一样"。

我们还顺手检查了"医院分配是否均衡"（会不会某个医院偏向收某一组病人），卡方 p=0.9997，**完全均衡**，排除了这个额外的干扰因素。

## 四、代码结构（方便接手 / 复用）

```
code/problem1_baseline.py
├── setup_cjk_font()        中文字体注册（画图用，避免出现方框乱码）
├── load_hospital_sheet()   读取单个医院的sheet，做基本清洗
├── onehot_to_category()    one-hot 4/3列 → 还原成1个分类变量，缺失记为"未知"
├── build_dataset()         合并两院数据，统一列名
├── split_age_by_group()    按组拆分年龄（去缺失），供描述统计/检验/画图共用
├── summarize_age()         年龄描述性统计（均值±SD、中位数、四分位数）
├── summarize_categorical() 分类变量描述性统计（各类别人数、占比）
├── compute_age_diagnostics() 年龄的Shapiro/Levene/ANOVA/Kruskal-Wallis统计量，供检验与画图共用
├── test_age()               年龄的正态性/方差齐性诊断 + ANOVA/Kruskal-Wallis自动选择 + 事后检验
├── test_categorical()       分类变量的卡方检验 + 蒙特卡洛置换兜底 + Cramer's V效应量
├── check_hospital_group_balance()  医院×组别分配均衡性核查
├── plot_age_box() / plot_categorical_stack() / plot_hospital_group_heatmap()  画图
└── main()                   串联以上所有步骤，打印摘要，保存表格和图
```

代码全程只用 `pandas / numpy / scipy / statsmodels / matplotlib`，用 `uv` 管理依赖（`pyproject.toml` + `uv.lock`），任何人 clone 下来在这个目录跑 `uv sync` 就能装好一样的环境，不用自己配。

## 六、写论文/汇报时可以直接用的话术

> 采用卡方独立性检验（分类变量）和 Kruskal-Wallis 秩和检验（年龄，因大样本下 Shapiro-Wilk 检验提示不满足正态性假设）对三组受试者的基线特征进行组间比较，并采用 Benjamini-Hochberg 法对多重比较进行校正。结果显示，三组在年龄（H=0.902, p=0.849）、既往抗抑郁药使用情况（χ²=0.877, p=0.928）、初始抑郁程度（χ²=4.744, p=0.849）方面均无统计学差异，提示三组基线特征基本均衡，具备可比性。婚姻状况经检验存在统计学差异（χ²=152.871, p<0.001），经列联表分析发现该差异主要由药物B组婚姻状况数据缺失比例较高（70/1050，其余两组该项无缺失）所致，而非三组患者婚姻构成本身存在实质差异，该缺失模式在后续分析中需作为数据局限性予以说明。

## 七、后续可能要注意的点（给做问题2/3的同学）

1. 婚姻状况的"未知"分布不均，如果问题2要分析"婚姻状况对疗效的影响"，B组这70人怎么处理需要提前商量好（排除 / 单独归一类 / 做敏感性分析），别几个人各做各的口径不一样。
2. 附件1里"抑郁程度"在二院有部分缺失（约42人），代码里同样按"未知"处理，问题2用到这个变量时要沿用同一套缺失值处理逻辑，保持前后一致。
3. 二院组1只有524人（比其他组少1人），属于原始数据的情况，不是我们哪步筛漏了，写报告时如果有人问起可以直接解释。
